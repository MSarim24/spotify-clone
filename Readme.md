# 📌 Introduction

Music streaming applications have become an essential part of digital entertainment systems, allowing users to access a large collection of music anytime and anywhere.

This project aims to develop a simplified music streaming web application inspired by platforms like Spotify.

The system allows users to:

- Stream songs
- Create and manage playlists
- Like songs
- Follow artists
- Browse albums
- Search content
- View artist profiles

The project focuses on implementing a relational database system along with a complete full-stack web application.

---

# ❗ Problem Statement

Modern music platforms are complex and require efficient data handling for:

- Users
- Songs
- Artists
- Albums
- Playlists
- Relationships between entities

Managing such interconnected data requires a well-designed relational database system.

This project addresses the need for a structured system where users can interact with music content efficiently while maintaining proper relationships between entities such as users, songs, artists, albums, and playlists.

---

# 🎯 Objectives

The main objectives of this project are:

- Design and implement a relational database for a music streaming system
- Develop a secure authentication system using JWT
- Allow users to perform core actions such as:
  - Liking and disliking songs
  - Following and unfollowing artists
  - Creating, updating, and deleting playlists
- Enable browsing of songs, albums, and artists
- Implement efficient API communication between frontend and backend
- Demonstrate proper use of database relationships:
  - One-to-Many
  - Many-to-Many

---

# 🌍 Scope of the Project

The scope of this project includes:
